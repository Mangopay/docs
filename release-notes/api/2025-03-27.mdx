## Pay by Bank payment method, leveraging open banking 

Platforms can now integrate Pay by Bank, which allows users to pay directly from their bank account via a range of bank wire transfer schemes, thanks to open banking regulations in Europe.

<CardGroup cols={2}>
<Card title="Guide" href="/guides/payment-methods/banking/pay-by-bank">
Pay by Bank
</Card>
</CardGroup>

The following endpoint has been added:

<CardGroup cols={2}>
<Card title="Endpoint" href="/api-reference/pay-by-bank/create-pay-by-bank-payin">
POST Create a Pay by Bank PayIn
</Card>
</CardGroup>

There is new webhook [event type](/webhooks/event-types) to indicate successful payment initiation: 

- `PAYIN_NORMAL_PROCESSING_STATUS_PENDING_SUCCEEDED`

There are new [functional errors](/guides/payment-methods/banking/pay-by-bank#functional-errors) (`ResultCode`) related to Pay by Bank:

- [206101 - RedirectURL page embedded](/errors/codes/206101)
- [206102 - RedirectURL already processed](/errors/codes/206102)
- [206103 - Bank not supported](/errors/codes/206103)
- [206104 - User not recognized](/errors/codes/206104)
- [206105 - Authentication method not supported](/errors/codes/206105)
- [206106 - Invalid user credentials](/errors/codes/206106)
- [206107 - Parallel session detected](/errors/codes/206107)
- [206108 - User blocked by bank](/errors/codes/206108)
- [206109 - User denied by bank](/errors/codes/206109)
