Name Description
authorized True if card charge authorization was successful or loan application was granted or another instance of transaction was successful, false otherwise. This flag tells us that the action the an user attempted succeeded or not.
authorization_aborted True if the transaction was aborted and authorization was not attempted, e.g. because of failed 3DS or second factor authentication.
captured True if capture was successful, false otherwise. If you do capture and authorize in a single request to PSP, you can skip this. It refers to card payments only.
refunded True if transaction was refunded to the customer.
chargebacked True if transaction was chargebacked. This flag should mean only chargebacks that you receive from your PSP or acquirer.
fraud_reported True if transaction was reported as fraud attempt by the customer, another fraud prevention solution, an analyst or a card scheme (TC-40 or SAFE, etc.). Please include in extra_data who reported this in a field source. Do not use this flag to inform us about chargebacks.
friendly_fraud True if transaction was reported as a friendly fraud. Use this flag only if you distinguish friendly frauds from other frauds. Otherwise use fraud_reported as a main fraud flag.
RFI True if the issuer requests more information about the transaction.
alert True if you received a notification of chargeback from e.g. Ethoca or Verifi.
3ds_attempted True if this transaction used 3D Secure authentication.
accepted True or false, indicating success or failure in SEPA payment
