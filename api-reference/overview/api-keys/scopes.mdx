sidebarTitle: Scopes

When you create an API key, you must assign it scopes and give it an alias to identify it. Each scope corresponds to endpoints that the key has permission to:
- Read, via GET calls
- Write, via POST and PUT calls

For more details about managing scopes, see the - [API keys](/api-reference/overview/api-keys) guide.

This page lists the endpoints in each scope. Each endpoint only belongs to one scope.
