#### Causes  

Usually, attempting to create a virtual account for a wallet with an unsupported currency returns a 400 error. In some rare cases, the creation attempt is asynchronous, and the resource is created with this 200 error.

#### What to do  

If this happens, retry the creation with a supported country value for the wallet currency, or create a new wallet in the relevant currency to be able to attach the virtual account to it.

For more information, see <a href="/guides/payment-methods/banking/virtual-account#iban-countries-and-currencies">IBAN countries and currencies</a>.

You can use the <a href="/api-reference/virtual-accounts/view-virtual-account-availabilities">GET View Virtual Account Availabilities</a> endpoint to see the virtual account currencies available for your platform.
