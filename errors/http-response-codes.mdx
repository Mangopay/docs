---
title: 'HTTP response codes'  
'og:description': 'Learn about HTTP response codes returned by Mangopay&apos;s REST API and what they mean.'  
---

The Mangopay API uses the following HTTP response codes to indicate the success or failure of a request.

<table class="html-table">
    <tbody>
        <tr>
            <th class="header">**Response code**</th>
            <th class="header">**Status**</th>
            <th class="header">**Description**</th>
        </tr>
        <tr>
            <td class="content">200</td>
            <td class="content">OK</td>
            <td class="content">Request successful.</td>
        </tr>
        <tr>
            <td class="content">204</td>
            <td class="content">No content</td>
            <td class="content">Request successful with no information returned.</td>
        </tr>
        <tr>
            <td class="content">400</td>
            <td class="content">string</td>
            <td class="content">Request impossible to process due to a logical error (e.g., missing parameter, invalid operation, constraint violation, etc.)</td>
        </tr>
        <tr>
            <td class="content">401</td>
            <td class="content">Unauthorized</td>
            <td class="content">Access to the API resource is not authorized due to an authentication failure (e.g., incorrect client ID, API Key, or URL).</td>
        </tr>
        <tr>
            <td class="content">403</td>
            <td class="content">Forbidden</td>
            <td class="content">Access to the API resource is forbidden, due to an authorization failure (e.g., deactivation of the account or a permission issue regarding a specific feature).</td>
        </tr>
        <tr>
            <td class="content">404</td>
            <td class="content">Not found</td>
            <td class="content">The requested resource cannot be found.</td>
        </tr>
        <tr>
            <td class="content">405</td>
            <td class="content">Method not allowed</td>
            <td class="content">HTTP method is not allowed (e.g., trying a GET instead of a POST).</td>
        </tr>
        <tr>
            <td class="content">409</td>
            <td class="content">Conflict</td>
            <td class="content">The request is not processed due to an identical [idempotency key](api-reference/idempotency).</td>
        </tr>
        <tr>
            <td class="content">411</td>
            <td class="content">Length required</td>
            <td class="content">The Content-Length header field is not defined and the server requires it.</td>
        </tr>
        <tr>
            <td class="content">413</td>
            <td class="content">Payload too large</td>
            <td class="content">The request entity is larger than the server is able to process (e.g., KYC document file larger than 10MB).</td>
        </tr>
        <tr>
            <td class="content">415</td>
            <td class="content">Unsupported media type</td>
            <td class="content">The format of the media sent is not supported by the server (e.g., wrong file format for KYC document file).</td>
        </tr>
        <tr>
            <td class="content">422</td>
            <td class="content">Unprocessable entity</td>
            <td class="content">The request cannot be fulfilled due to semantic errors.</td>
        </tr>
        <tr>
            <td class="content">429</td>
            <td class="content">Too many requests</td>
            <td class="content">Too many requests have been sent in a given period of time ([rate limiting](api-reference/rate-limiting)).</td>
        </tr>
        <tr>
            <td class="content">500</td>
            <td class="content">Internal server error</td>
            <td class="content">We invite you to try again later or to contact Support [via the Hub](https://hub.mangopay.com/) if the problem persists.</td>
        </tr>
    </tbody>
</table>
